# Certification & Recall Practice

A static web app for self-testing on certification topics and book recall. Supports multiple providers (Snowflake, AWS, etc.) with auto-generated test manifests and per-question feedback.

## Features

- Multiple-choice practice tests (single-select and multi-select)
- Scenario-based and direct recall questions
- Immediate per-question feedback with explanations for every option
- Domain-level score analysis on completion
- PDF report export
- Session persistence (resume where you left off)
- Two question generation modes:
  - **Certification** — derived from public documentation for exam prep
  - **Recall** — derived from book chapters for retention testing (shows a disclaimer banner)

## Running Locally

The app is purely static (HTML/CSS/JS + JSON files). You just need a local server to avoid CORS issues when fetching JSON.

### Python (built-in)

```bash
# Python 3
python3 -m http.server 8000

# Then open http://localhost:8000
```

### Node.js

```bash
# Using npx (no install needed)
npx serve .

# Or install globally
npm install -g serve
serve .

# Then open http://localhost:3000
```

### VS Code

If you use the Live Server extension, right-click `index.html` → "Open with Live Server".

## Adding a New Test

### Certification questions (from public docs)

1. Provide documentation/notes to the agent using `generate_exam_questions`.
2. Save as `test_N.json` in the appropriate `<provider>/<topic>/` folder.
3. Push to `main` — GitHub Actions regenerates `manifest.json`.

### Recall questions (from books/private material)

1. Place the source PDF in `<provider>/<topic>/sources/` (gitignored).
2. Provide the content to the agent using `generate_recall_questions`.
3. Save as `test_N.json` in the topic folder.
4. The topic folder must contain a `DISCLAIMER.md` and `sources/README.md` (created automatically by the prompt).
5. Push to `main` — GitHub Actions regenerates `manifest.json`.

The recall test JSON includes `"questionType": "recall"` which triggers a disclaimer banner in the web interface.

## Manifest Generation

The manifest is auto-generated by `.github/scripts/generate_manifest.py`. It scans all `<provider>/<topic>/test_N.json` files and builds the index.

To regenerate locally:

```bash
python3 .github/scripts/generate_manifest.py
```

## Coverage Tracking

Recall tests include a `coverage` object at the end of the JSON:

```json
"coverage": {
  "areas_covered": ["..."],
  "areas_not_covered": ["..."]
}
```

Use `areas_not_covered` to guide the next batch of questions from the same source material.

## Hosting

The app is deployed via GitHub Pages from the `main` branch. Since source PDFs are gitignored and recall questions don't reproduce copyrighted text, the public repo is safe.
